# GLM-5：从Vibe编码到智能体工程

## 摘要

我们提出了 GLM-5，这是一个旨在将 Vibe 编码范式过渡到智能体工程的新一代基础模型。GLM-5 基于其前代产品的智能体、推理和编码 (ARC) 能力，并采用动态智能体算法 (DSA) 来显著降低训练和推理成本，同时保持长上下文保真度。为了提升模型对齐和自主性，我们实现了一种新的异步强化学习基础设施，通过将生成与训练解耦，大幅提高了训练后的效率。此外，我们还提出了新的异步智能体强化学习算法，进一步提升了强化学习的质量，使模型能够更有效地从复杂的长时程交互中学习。凭借这些创新，GLM-5 在主要的开放基准测试中取得了最先进的性能。最重要的是，GLM-5 在实际编码任务中展现了前所未有的能力，在处理端到端软件工程挑战方面超越了以往的基线模型。代码、模型和更多信息可在https://github.com/zai-org/GLM-5获取。

## 引言

追求通用人工智能（AGI）不仅需要扩展模型参数，还需要从根本上重新思考智能效率和自主改进的架构。随着GLM-4.5的发布，我们证明了将智能体、推理和编码（ARC）能力整合到单一的专家模型（MoE）架构中，可以在各种基准测试中获得最先进的结果。然而，随着大型语言模型（LLM）从被动的知识库向主动的问题求解器转变，计算成本和实际应用适应性的双重挑战——尤其是在复杂的软件工程中——已成为主要的瓶颈。

我们隆重推出 GLM-5，这是我们为克服这些障碍而设计的下一代旗舰模型。GLM-5 在性能和效率方面实现了范式转变，在包括 ArtificialAnalysis.ai、LMArena Text 和 LMArena Code 在内的主要公开排行榜上均取得了领先地位。更重要的是，GLM-5 重新定义了实际编码的标准，展现了前所未有的能力，能够处理复杂的端到端软件开发任务，远远超出了 SWE-bench 等传统静态基准测试的范围。

GLM-5 在人工智能指数 v4.0 中获得 50 分，成为新的开放权重模型领跑者（参见图 2），较 GLM-4.7 的 42 分大幅提升 8 分，这主要得益于其在智能体性能和知识/幻觉方面的改进。这是开放权重模型首次在人工智能分析智能指数 v4.0 中获得 50 分。

由加州大学伯克利分校发起的LMArena是一个透明的共享平台，它通过数百万个真实任务（包括写作、编码、推理、设计、搜索和创作）来评估和比较前沿人工智能的能力，并结合人类的判断。大量的人机交互能够产生反映真实世界实用性的信号，这使其区别于其他静态基准测试。图 3显示，GLM-5在文本竞技场和代码竞技场中再次位列开源模型榜首，并且总体性能与Claude-Opus-4.5和Gemini-3-pro不相上下。

智能体的长期一致性变得越来越重要。编码智能体现在可以自主编写代码数小时，人工智能模型能够完成的任务范围和时长也可能不断扩大。我们使用两个基准测试：Vending-Bench 2 和 CC-Bench-V2，来评估 GLM-5 完成长期任务的能力。Vending-Bench 2 是一个用于衡量人工智能模型在长期运营业务方面性能的基准测试。模型的任务是运营一个模拟的自动售货机业务一年，并根据其最终的银行账户余额进行评分。图 4（左）显示，GLM-5 在所有开源模型中排名第一，最终账户余额为 4,432 美元。它接近 Claude Opus 4.5 的水平，展现出强大的长期规划和资源管理能力。图 4（右）进一步展示了我们内部评估套件 CC-Bench-V2 的结果。 GLM-5 在前端、后端和长期任务中明显优于 GLM-4.7，缩小了与 Claude Opus 4.5 的差距。

方法。图 5展示了 GLM-5 的整体训练流程。我们的基础模型训练始于一个庞大的 27 万亿词元语料库，早期重点训练代码和推理能力。随后，我们采用了一个独立的训练中期阶段，逐步将上下文长度从 4K 扩展到 200K，尤其关注长上下文的智能体数据，以确保模型在复杂工作流程中的稳定性。在训练后阶段，我们超越了标准的 SFT（结构化框架训练）。我们实现了一个顺序强化学习流程——首先是推理强化学习，然后是智能体强化学习，最后是通用强化学习。至关重要的是，我们在整个过程中都采用了策略内跨阶段蒸馏技术，以防止灾难性遗忘，确保模型在保持其敏锐推理能力的同时，也能成为一个强大的通用模型。总而言之，GLM-5 性能的飞跃得益于以下技术贡献：

首先，我们采用 DSA（DeepSeek 稀疏注意力）  [9GLM- 5采用了一种全新的架构创新，显著降低了训练和推理成本。GLM-4.5 通过标准的 MoE 架构提高了效率，而 DSA 则允许 GLM-5 基于词元重要性动态分配注意力资源，从而大幅降低计算开销，同时又不影响长上下文理解或推理深度。借助 DSA，我们将模型参数扩展到 7440 亿，并将训练词元预算扩展到 28.5 万亿个词元。

其次，我们构建了一个全新的异步强化学习基础设施。基于“slime”框架和GLM-4.5中初始化的解耦式rollout引擎，我们的新基础设施进一步将生成与训练解耦，从而最大限度地利用GPU。该系统能够大规模探索智能体轨迹，避免了以往阻碍迭代速度的同步瓶颈，显著提高了强化学习后训练流程的效率。

第三，我们提出了一系列新型异步智能体强化学习算法，旨在提升自主决策的质量。在GLM-4.5中，我们利用迭代自蒸馏和结果监督来训练智能体。对于GLM-5，我们开发了异步算法，使模型能够持续地从多样化的、长时程的交互中学习。这些算法经过专门优化，能够提升模型在动态环境中的规划和自纠错能力，从而直接促成了我们在实际编码场景中的优势。

最后，GLM-5的另一项技术贡献在于，它从一开始就针对中国GPU生态系统进行了全栈适配。我们已成功完成了对包括华为昇腾、Moore Threads、海光、寒武纪、昆仑鑫、MetaX和Enflame在内的七大国内主流芯片平台的深度优化，涵盖了从底层内核到上层推理框架的各个层面。

凭借这些进步，GLM-5 不仅是一个更强大的模型，而且为下一代人工智能代理提供了更高效、更实用的基础。我们向社区发布 GLM-5，以进一步推进高效、自主通用智能的发展。

## 预训练

与 GLM-4.5 类似，GLM-5 的基础模型也经历两个阶段：预训练阶段用于培养通用语言和编码能力，中期训练阶段用于培养智能体和长上下文能力。我们扩展了 GLM-5 所有训练阶段的训练令牌预算，基础模型的总令牌数达到 28.5 万亿。

### 模型结构

模型尺寸缩放。GLM-5 可扩展至 256 位专家，并将层数减少到 80 层，以最大限度地减少专家并行通信开销。这使得模型参数量达到 744 亿（其中 400 亿为活跃参数），是 GLM-4.5 总参数量（355 亿，其中 320 亿为活跃参数）的两倍。

多重潜注意力机制。通过采用简化的键值向量，多潜在注意力机制（MLA）  [24]与分组查询注意力 (GQA) 的效果相当，但能更好地节省 GPU 内存，并能更快地处理长上下文序列。

然而，在我们使用 Muon 优化器的实验中，我们发现具有 576 维潜在键值缓存的 MLA 的性能无法与具有 8 个查询组的 GQA（记为 GQA-8，具有 2048 维键值缓存）的性能相媲美。为了克服性能差距，我们提出了一种对 GLM-4.5 中 Muon 优化器方法的改进方案。在原始方法中，我们对上投影矩阵应用了矩阵正交化。

对于多头查询，键和值并非直接对应，而是被分割成更小的矩阵，分别对应不同的注意力头，并对这些独立的矩阵进行矩阵正交化。这种方法被称为 Muon Split，它使得不同注意力头的投影权重能够在不同的尺度上更新。如表1所示，该方法有效地提升了 MLA 的性能，使其与 GQA-8 的性能相当。在实践中，我们还发现，使用 Muon Split 后，GLM-5 的注意力 logits 的尺度在预训练期间保持稳定，无需任何裁剪策略。

MLA的另一个缺点是解码过程中计算成本高。在解码过程中，MLA执行576维点积运算，高于GQA的128维计算。而DeepSeek-V3中的注意力头数量是根据H800的屋顶线选择的 [60因此，它不适用于其他硬件。考虑到 MLA 在训练和预填充过程中采用多头注意力（MHA）风格，我们将注意力头维度从 192 增加到 256，并将注意力头的数量减少 1/3。这样既保持了训练计算量和参数数量不变，又降低了解码计算量。该变体（如表1中的 MLA-256 所示）在缪子分裂问题下的性能与 MLA 相当。

基于参数共享的多标记预测。多标记预测（MTP）  [13；25]提高了基础模型的性能，并可作为推测性解码的草稿模型 [20然而，在训练过程中，为了预测下一个词元，需要使用 MTP 层。因此，MTP 参数和键值缓存的内存使用量与推测步骤的数量呈线性关系。而 DeepSeek-V3 仅使用单个 MTP 层进行训练，并在推理过程中预测接下来的两个词元。训练和推理结果之间的差异会降低第二个词元的接受率。因此，我们提出在训练过程中共享 3 个 MTP 层的参数。这样既能保持草稿模型的内存开销与 DeepSeek-V3 一致，又能提高接受率。表 2显示，在我们的私有提示集上，当推测步骤数量相同（4）时，GLM-5 的接受长度比 DeepSeek-V3.2 更长。

#### 使用 DeepSeek 稀疏注意力 (DSA) 继续进行预训练

我们在培训中使用DSA。DSA的核心理念是 [9是为了取代传统的密集型注意力——在以下情况下会变得极其昂贵：128K上下文——采用动态、细粒度的选择机制。与固定模式（例如滑动窗口）不同，DSA 会“查看”内容以决定哪些标记是重要的。从研究者的角度来看，DSA 特别引人注目之处在于它是如何通过从密集基础模型进行持续预训练而引入的。这避免了从头开始训练的“天文数字”成本。过渡遵循两阶段的“密集预热和稀疏训练自适应”策略。DeepSeek-V3.2-Exp 保持了与其密集前代相同的基准性能，证明了长上下文中 90% 的注意力条目确实是冗余的。对于长序列，DSA 将注意力计算量减少了大约 1.5-2 倍，这对于我们正在构建的推理密集型智能体来说至关重要，因为它能够以一半的 GPU 成本处理 128K 个上下文。

DSA 训练从中期训练结束时的基础模型开始。预热阶段包含 1000 个步骤，每个步骤使用 14 个序列（每个序列包含 202,752 个 token）进行训练，最大学习率为 5e-3。稀疏自适应阶段使用中期训练的训练数据和超参数，并处理 200 亿个 token。虽然训练预算远小于 DeepSeek-V3.2（9437 亿个 token），但我们发现这足以使 DSA 模型自适应到与原始 MLA 模型性能相当的水平。如表 3所示， DSA 模型的长上下文性能与 MLA 模型接近。为了进一步验证 DSA 训练的有效性，我们分别使用相同的 SFT 数据集对 DSA 和 MLA 模型进行微调，发现两个模型在训练损失和评估基准方面表现一致。

#### 有效注意力变异体的消融研究

超越DSA  [26，我们探索了几种基于 GLM-9B 的替代高效注意力机制，基线模型在所有 40 层中都采用了分组查询注意力机制，并使用 128K 个 token 的上下文窗口进行了微调。我们评估了以下方法：
- 滑动窗口注意力（SWA）交错：在整个网络中均匀应用全注意力层和窗口注意力层的固定交替模式。
- 门控 DeltaNet (GDN)  [54]：一种线性注意力变体，它用门控线性递归代替二次 softmax 注意力计算，将注意力的计算成本从序列长度的二次方降低到线性。
在此基础上，我们提出两项改进：
- SWA模式（基于搜索）：受PostNAS启发 [15本文提出了一种基于搜索的自适应方法，该方法能够识别用于SWA转换的最佳层子集，同时保持其余层的完整注意力机制。我们采用束搜索策略来确定能够最大化长上下文下游任务性能的配置。为了降低计算成本，我们仅在16K上下文长度下进行搜索，并将结果模式推广到所有其他输入长度。具体来说，我们使用8的束大小，每步优化两层；对于GLM-9B（40层），该过程大约需要10步即可收敛。在每一步中，候选模式都会在RULER基准测试集上进行评估 。17在 16K 上下文长度下，保留前 8 个候选模式用于后续步骤。最终得到的模式为SFSSFFSSSFFFFSSFSFFFFFFSFSFSSFSSFSFSSFSSS，其中S和F分别表示 SWA 层和全注意力层。如表 4所示，这种基于搜索的配置显著优于固定交错方法。值得注意的是，尽管该模式仅在 16K 上下文长度下进行了优化，但它展现出良好的长度泛化能力，在所有测试的上下文长度下均保持有效。
- SimpleGDN：一种极简的线性化策略，旨在最大程度地复用预训练权重，并在 GDN 的基础上进行改进，以实现持续训练的自适应。我们完全移除了Conv1d和显式门控模块，而是将预训练的查询、键和值投影权重直接映射到线性递归公式中。这种简化消除了对额外参数的需求，同时保留了线性注意力机制的效率优势。

我们在四个长上下文基准测试中评估所有方法：RULER  [17]，MRCR，以及 RepoQA。我们使用 190B 个 token 和 64K 的上下文长度持续训练每种方法，并保持高效注意力层和完整注意力层 1:1的比例。对于 GDN 和 SimpleGDN 方法，我们遵循 Jet-Nemotron  [15]管道。

表5的结果 揭示了高效注意力方法之间清晰的权衡层级关系。简单地交错滑动窗口注意力（SWA）会导致长上下文任务（例如，在 RULER@128K 数据集上，精度差距为 30.35 分。而基于搜索的层选择通过在关键层保留完整的注意力机制，显著缩小了这一差距。诸如 GDN 之类的线性注意力机制变体进一步提升了精度，但代价是需要额外的参数；SimpleGDN 通过最大程度地复用预训练权重，实现了最佳平衡。然而，所有这些方法在细粒度检索任务上都存在固有的精度差距——在 RULER@128K 数据集上高达 5.69 分，在 RepoQA@128K 数据集上高达 7.33 分——这是由于在持续训练适应过程中，即使一半的层保留了完整的注意力机制，高效的注意力机制也不可避免地会造成信息损失。相比之下，DSA 从设计上来说是无损的：其闪电索引器在不丢弃任何长程依赖关系的情况下实现了词元级稀疏性，使其能够应用于所有层而不会降低精度。

## 预训练数据

网络。基于 GLM-4.5 数据管道，我们改进了针对海量网络数据集的选择标准。我们引入了另一个 DCLM  [21我们开发了一种基于句子嵌入的分类器，用于识别和聚合标准分类器无法获取的高质量数据。为了应对长尾知识的挑战，我们利用了世界知识分类器（该分类器通过维基百科条目和LLM标注数据进行了优化），从原本质量中等偏低的数据中提取有价值的信息。

代码。我们利用来自主流代码托管平台的最新快照和更大规模的包含代码的网页集合，扩展了代码预训练语料库，从而使模糊去重后的唯一词元数量增加了 28%。为了提高语料库的完整性并降低噪声，我们修复了 Software Heritage 代码文件中的元数据对齐问题，并采用了更精确的语言分类流程。我们遵循 GLM-4.5 的质量感知采样策略来处理源代码和与代码相关的网页文档。此外，我们还针对更广泛的低资源编程语言（例如 Scala、Swift、Lua 等）训练了专用分类器，从而提高了这些语言的采样质量。

数学与科学。我们从网页、书籍和论文中收集高质量的数学和科学数据，以进一步提升推理能力。具体而言，我们改进了网页内容提取流程和书籍、论文的PDF解析机制，以提高数据质量。我们采用大型语言模型对候选文档进行评分，并仅保留最具教育意义的内容。对于长上下文文档，我们开发了一种分块聚合评分算法，以提高评分准确率。我们实施了过滤流程，严格避免使用合成数据、AI生成数据或基于模板的数据。

## 中期训练

在 GLM-4.5 中引入的中期训练框架的基础上，我们在 GLM-5 中扩大了训练量和最大上下文长度，以进一步增强模型的推理能力、长上下文能力和智能能力。

扩展的背景和训练规模。我们将上下文窗口逐步扩展，共分为三个阶段：32K（1T 个词元）、128K（500B 个词元）和 200K（500B 个词元）。与 GLM-4.5 中 128K 的最大值相比，新增的 200K 阶段显著提升了模型处理超长文档和复杂多文件代码库的能力。在后续阶段，长文档和合成代理轨迹会相应地进行上采样。

软件工程数据。我们保留了将仓库级代码文件、提交差异、GitHub 问题、拉取请求和相关源文件连接成统一训练序列的范式。在 GLM-5 中，我们放宽了仓库级过滤标准，扩大了符合条件的仓库范围，从而获得了约 1000 万个问题-拉取请求对，同时加强了单个问题级别的质量过滤以降低噪声。此外，我们还为每个问题-拉取请求对检索了更多相关文件，从而获得了更丰富的开发上下文和更广泛的真实软件工程场景覆盖。过滤后，数据集中问题-拉取请求部分包含约 1600 亿个唯一标记。

## 训练基础设施

### 内存效率

灵活的 MTP 放置。采用交错式流水线并行 [31模型组件可以灵活地分配到各个阶段。MTP 模块包含嵌入、 Transformer和输出组件。与其他模块相比，它的内存占用显著更高，导致阶段级内存不平衡。为了实现参数共享，我们将 MTP 的输出层与主输出层置于最终阶段，同时将其嵌入和 Transformer 组件放置在前一个阶段。这降低了最终阶段的内存压力，并改善了流水线各阶段之间的内存平衡。

ZeRO2 流水线梯度分片。每个流水线等级维护多个阶段 [31]，并且简单地说，每个阶段都需要一个完整的梯度缓冲区来进行累积和优化器更新。受 ZeRO2 的启发 [38为了解决这个问题，我们将梯度分片到数据并行的进程中，使得每个阶段仅存储完整梯度的 1/dp 部分。此外，我们每次只保留两个阶段的完整累积缓冲区，并通过双缓冲机制重用它们。当一个阶段的缓冲区累积连续微批次的梯度时，前一个阶段缓冲区的梯度同步是并行执行的。这使得持久梯度内存减少到每个阶段的分片缓冲区加上两个用于滚动累积的完整缓冲区，而实际上并没有额外的同步开销。

Muon分布式优化器的零冗余通信。传统的Muon实现会在每个数据并行进程上收集所有模型参数，导致瞬态内存峰值和冗余通信。我们将收集操作限制在每个进程拥有的参数分片上，并将本地计算与分片通信重叠。这消除了冗余通信，并显著降低了与优化器相关的峰值内存开销。

流水线激活卸载。在流水线预热期间，前向执行会先于反向传播，从而延长中间激活的生命周期。我们在前向执行后将激活卸载到主机内存，并在反向执行之前重新加载它们 。58为了进一步降低峰值内存使用量，卸载操作以层级粒度进行。结合细粒度的重新计算，这大大减少了将激活值驻留在 GPU 内存中的必要性。卸载和重新加载的调度与计算任务重叠，同时避免与点对点通信和 MoE 令牌路由（分发和组合）发生争用。这在几乎零开销的情况下，显著降低了激活值的内存占用。

为了降低峰值内存占用，我们采用了序列分块输出投影方法。 输出投影和交叉熵损失函数会因存储反向传播所需的激活值以及在损失计算过程中将其提升至更高精度而产生瞬态内存开销。为了降低这种开销，我们将输入序列分割成更小的块，并在每个块上独立计算投影和损失，在完成前向和反向传播后释放激活值，然后再进行下一个块的计算。因此，随着块数的增加，峰值内存使用量会降低。在合适的块数下，这种方法可以缓解输出层的内存压力，同时保持与非分块执行相当的性能。

### 并行效率

高效的延迟权重梯度计算。为了减少管道气泡，我们延迟计算关键路径上的一些权重梯度 [37] . 细粒度延迟以及优化的存储和通信重叠可提高吞吐量，同时将内存开销控制在一定范围内。

高效的长序列训练。较长的序列会加剧数据并行组和流水线并行组之间的负载不均衡。我们通过感知工作负载的序列重排序、动态重新分配注意力计算以及将数据并行排序灵活地划分为不同大小的上下文并行组来解决这个问题 。12；47] . 为 QKV 张量实现分层全对全重叠的节点内和节点间通信，以减少延迟。

### INT4 量化感知训练

为了在低精度下提供更高的准确率，我们在SFT阶段应用了INT4 QAT。此外，为了进一步降低训练时间开销，我们开发了一种量化核，该量化核既适用于训练，也适用于离线权重量化，从而确保训练和推理过程中的位级一致性。

# 后训练

GLM-5 的训练后阶段旨在将基础模型转化为具备强大推理、编码和智能能力的智能助手。如图 5所示，我们的流程遵循渐进式对齐策略：首先进行多任务监督微调 (SFT)，引入复杂的交错思维模式；随后进行专门的强化学习 (RL) 阶段，用于推理和智能任务；最后进行通用强化学习阶段，实现类人风格的对齐。通过利用策略内跨阶段蒸馏作为最终的优化步骤，GLM-5 能够有效缓解能力退化，同时充分利用每个训练阶段的性能提升。

# 监督式微调

与 GLM-4.5 相比，GLM-5 在 SFT 阶段显著扩展了Agent和Coding数据的规模。GLM-5 的 SFT 语料库涵盖三大类：
• 综合聊天：问答、写作、角色扮演、翻译、多轮对话和长篇互动；
• 推理能力：数学推理、编程推理和科学推理；
• 编码与代理：前端和后端工程代码、工具调用、编码代理、搜索代理和通用代理。
此外，GLM-5 在 SFT 期间将最大上下文长度扩展至 202,752 个词元。该模型除了更新聊天模板外，还支持三种不同的思维特征（见图7），包括：
- 交错式思考：该模型在每次响应和工具调用之前都会进行思考，从而提高指令遵循率和任务的质量。
- 思维保留：在编码代理场景中，该模型会自动保留多轮对话中的所有思维模块，重用现有推理过程，而不是从头开始重新推导。这减少了信息丢失和不一致性，非常适合长期、复杂的任务。
- 回合级思考：该模型支持对会话中的推理进行逐回合控制——对于轻量级请求禁用思考以减少延迟/成本，对于复杂任务启用思考以提高准确性和稳定性。

GLM-5 通过在行动之间进行思考并保持回合间的一致性，在复杂任务中实现了更稳定、更可控的行为。

对于通用聊天任务，我们优化了回复风格，使其比 GLM-4.5 更符合逻辑且简洁。对于角色扮演任务，我们收集并构建了一个涵盖多种语言和角色配置的更广泛、更多样化的数据集。具体而言，我们定义了几个评估维度——包括指令遵循度、语言表达能力、创造性、逻辑连贯性和长对话一致性——并结合自动和人工筛选来整理和优化数据。

对于推理任务，我们进一步增强了模型的推理深度。具体而言，对于逻辑推理，我们构建可验证的问题，并使用拒绝抽样法合成高质量数据。对于数学和科学问题，我们应用基于难度的筛选过程，仅保留对 GLM-4.7 模型具有挑战性的问题。

对于编码和智能体任务，与 GLM-4.5 相比，GLM-5 构建了大量的执行环境以获得高质量的轨迹，尤其侧重于真实场景和长周期任务。我们进一步利用专家强化学习和拒绝采样改进了 SFT 数据集。轨迹中的错误片段被保留，但在损失函数中被屏蔽，这使得模型能够在不强化错误行为的情况下学习纠错行为。

## 推理强化学习

强化学习算法。我们的强化学习算法建立在GRPO之上 [40并采用了 IcePop 技术 [61为了缓解训练-推理不匹配问题，即强化学习优化过程中推理分布与训练分布之间的差异，我们明确区分了训练策略。与原始 IcePop 公式相比，我们移除了 KL 正则化项以加速强化学习的改进。

DSA RL。我们对基于 DSA 架构的模型进行了大规模强化学习 (RL) 训练。与 MLA 相比，DSA 引入了一个额外的索引器，用于检索前 k 个最相关的键值对，并在检索到的子集上稀疏地计算注意力。检索到的前 k 个结果对于强化学习的稳定性至关重要。这类似于 MoE 模型如何使用路由重放 [62为了保留已激活的前 k 个专家，以确保训练和推理的一致性。然而，直接将此策略应用于索引器重放，即在每个标记位置存储索引器的前 k 个索引，显然是不切实际的，存储所有这些索引会产生巨大的存储成本，以及训练引擎和推理引擎之间大量的通信开销。

我们发现，采用确定性的 top-k 算子能够有效解决 DSA 索引器 token 选择中的训练-推理不匹配问题。与 SGLang 的 DSA 索引器中使用的基于 CUDA 的非确定性 top-k 实现相比，直接使用简单的torch.topk虽然速度稍慢，但具有确定性。它能够产生更一致的输出，并带来显著的强化学习 (RL) 性能提升。相比之下，其他非确定性的 top-k 算子（例如，CUDA 或 TileLang 实现）在强化学习过程中仅经过几步后就会导致性能急剧下降，同时熵值也急剧降低。因此，在我们的强化学习阶段，我们将torch.topk作为训练引擎中 DSA 索引器的默认 top-k 算子。此外，我们在强化学习过程中默认冻结索引器参数，以加速训练并防止索引器出现不稳定的学习过程。

混合领域推理强化学习。在推理强化学习阶段，我们对四个领域进行混合强化学习训练：数学、科学、代码和工具集成推理（TIR）。对于数学和科学，我们从开源数据集中收集数据 [10；30并与外部标注供应商共同开发数据集。我们进一步应用难度过滤，将训练重点放在 GLM-4.7 很少能正确解决或总是失败的问题上，而这些问题仍然可以被更强大的教师模型（例如 GPT-5.2 xhigh 和 Gemini 3 Pro Preview）解决。对于代码，我们涵盖了竞赛编程风格的任务和科学编程任务。前者主要来自 Codeforces 和 TACO 等代表性数据集 。23]和 SYNTHETIC-2-RL  [35而后者则是通过将问题分解为实现正确答案所需的最小代码实现，从内部问题库中构建而成。对于TIR，我们重用了数学和科学强化学习数据中更具挑战性的子集，并与标注供应商合作构建STEM问题，这些问题专门设计用于使用外部工具作答。在强化学习训练过程中，我们分配特定领域和来源的评判模型或评估系统，以产生二元结果奖励。我们保持四个领域之间的整体混合比例大致平衡，并在混合强化学习设置下，在每个领域都持续观察到稳定且显著的提升。

## 智能体强化学习

为了提升 GLM-5 的智能体性能，我们开发了一个完全异步且解耦的强化学习框架，并针对编码和搜索智能体任务优化了 GLM-5。传统的同步强化学习在长时域智能体部署过程中会面临严重的 GPU 空闲问题。通过中央多任务部署协调器解耦推理引擎和训练引擎，我们实现了跨多种智能体工作负载的高吞吐量联合训练。

为了在异步离策略条件下保持训练稳定性，我们引入了两种关键机制。首先，Token-in-Token-out (TITO) 网关通过保留精确的动作级对应关系来消除重新标记化不匹配。其次，我们采用直接双边重要性采样，它应用了一种标记级裁剪机制。为了实现对数概率的展开，同时有效控制离策略偏差而无需跟踪历史策略检查点，我们还采用了一种动态规划感知路由，以在长上下文推理期间最大化键值缓存的重用，从而加速大规模多属性模型的运算。为了扩展智能体环境，我们在三个领域扩展了可验证的训练环境：超过 1 万个真实世界的软件工程 (SWE) 任务、终端任务和高难度多跳搜索任务。有关智能体强化学习的更多详细信息，请参见后续的第 4 节。

## 通用强化学习

多维优化目标。我们将通用强化学习的优化目标分解为三个互补的维度：基础正确性、情感智能和特定任务质量。

基础正确性维度是响应质量的基石。它旨在纠正各种影响模型输出可用性的错误类型，包括指令执行错误、逻辑不一致、事实错误、知识错觉和语言表达不流畅。目标是最大限度地降低错误率，使响应达到可用的基准水平。我们认为这是所有后续优化的先决条件：无论响应看起来多么完美，包含事实错误或误解用户意图的内容都可能误导用户。

情商维度在核心正确性之外，进一步优化了用户体验。它旨在生成富有同理心、洞察力强且风格接近自然人际沟通的回复，使用户与模型的互动更加自然流畅，更具吸引力。

任务特定质量维度旨在针对各种特定任务进行精细化优化。它以基础正确性所建立的可用性为基础，力求在每个任务类别中将答案从仅仅正确提升到真正的高质量。该维度涵盖广泛的任务，包括写作、文本处理、主观和客观问答、角色扮演和翻译。每个任务领域都需要不同的奖励信号，因此需要一个混合奖励系统。

混合奖励系统。为了监督上述多样化的目标，我们构建了一个混合奖励系统，该系统整合了三种互补的奖励信号：基于规则的奖励函数、结果奖励模型（ORM）和生成式奖励模型（GRM）。每种模型都有其独特的优势和劣势，它们的结合是实现稳定、高效且可扩展的通用强化学习训练过程的关键。

基于规则的奖励机制能够提供精确且易于解释的信号，但其作用范围仅限于可以用确定性规则表达的方面。基于规则的奖励机制（ORM）能够提供低方差的信号和较高的训练效率，但更容易受到奖励机制攻击，即策略利用表面模式而非真正提升核心能力。基于规则的奖励机制（GRM）利用语言模型生成标量或结构化的评估结果，对这类攻击更具鲁棒性，但往往表现出更高的方差。通过融合这三种信号类型，我们获得了一种能够平衡精确性、效率和鲁棒性的奖励系统，从而弥补了任何单一组件的不足。

人机协作式对齐。我们通用强化学习流程的一个显著特点是明确地融入了高质量的人工撰写的回复。我们并非仅仅依赖模型生成的回复，而是引入专家撰写的回复作为风格和质量的参考标准。这是基于这样的观察：纯粹由模型生成的优化往往会趋向于明显的“模型式”模式——通常冗长、公式化，或者缺乏熟练人类写作的细微差别。通过让模型接触人工撰写的范例，我们鼓励它采用更自然、更符合人类习惯的回复模式。

## Policy 跨阶段蒸馏

在我们的多阶段强化学习流程中，针对不同目标进行顺序优化会导致先前获得的能力累积退化。为了缓解这个问题，我们在最后阶段执行策略内跨阶段蒸馏，采用策略内蒸馏算法 [14；52；51；28为了快速恢复在早期SFT和RL阶段（推理RL和通用RL）获得的技能，具体来说，先前训练阶段的最终检查点用作教师模型，训练提示从相应教师的RL训练集中采样，并按适当比例混合。

## 强化学习训练基础设施：slime框架

我们继续使用 slime 作为 GLM-5 的统一后训练基础设施，从而实现大规模端到端强化学习 (RL)。GLM-5 没有引入新的系统组件，而是充分利用slime 的各项功能，(1) 通过自由形式的部署定制和基于服务器的执行模型扩展任务覆盖范围；(2) 通过混合精度训练/部署以及 MTP 和预填充解码 (PD) 分解显著提高吞吐量，尤其适用于多轮强化学习工作负载；(3) 通过心跳驱动的部署容错和路由器级服务器生命周期管理提高鲁棒性。

### 扩展：通过高度可定制的部署实现灵活培训

GLM-5 的训练后目标涵盖范围广泛。为了支持这种多样性而无需针对特定任务创建分支，GLM-5 利用了 slime 高度可定制的部署界面及其基于服务器的部署执行机制。

高度可定制的部署。slime提供了一个灵活的接口，用于实现特定任务的部署逻辑，包括多轮交互循环、工具调用、环境反馈处理和验证器引导的分支，而无需修改底层基础架构。GLM-5 利用此功能，在一个统一的训练堆栈中支持广泛的领域和训练范式，包括但不限于推理强化学习、通用强化学习、智能体强化学习和策略蒸馏。

通过 HTTP API 实现基于服务器的部署。slime通过标准的 HTTP API 公开其部署服务器和推理路由器，使用户能够以与传统推理引擎相同的方式与 slime 的服务层进行交互。这使得部署逻辑与训练过程的边界解耦：外部代理框架和环境可以直接调用服务器/路由器端点，而优化后端对于短期单轮训练和长期多轮轨迹训练均保持不变。

### 规模化：强化学习部署的尾延迟优化

对于强化学习部署而言，优化目标并非总吞吐量，而是端到端延迟，而端到端延迟主要取决于每个步骤中最慢（长尾）的样本。实际上，单个滞后的轨迹就可能导致同步点（例如，批处理完成、缓冲区就绪、训练器更新）停滞，并直接影响实际运行时间。因此，GLM-5 充分利用了 slime 的面向延迟的服务和调度机制，以最大限度地降低中值延迟，更重要的是，降低尾部延迟。

GLM-5 采用基于 DP-attention 的多节点推理实现无队列服务，以应对 MLA 的挑战。 为了避免排队延迟，即使在突发流量的情况下，也必须及时响应部署请求，这需要大量的键值缓存容量。GLM-5 采用多节点推理部署（例如，在 8 个节点上部署 EP64 和 DP64）来提供充足的分布式键值缓存。引入 DP-attention 的主要目的是防止跨不同 rank 复制键值。

利用 FP8 展开和 MTP 降低尾延迟。GLM -5 使用 FP8 进行展开推理，以降低每个标记的延迟并缩短长轨迹的完成时间。此外，GLM-5 还利用了 slime 对多标记预测 (MTP) 的支持，这在强化学习 (RL) 展开中常见的小批量解码机制下尤为有效。由于尾延迟通常由小批量延迟（例如，罕见的长上下文、复杂的多轮推理、工具密集型轨迹）驱动，MTP 在长尾上带来了不成比例的显著优势，改善了最慢样本的完成时间，从而减少了步级停顿时间。

为了防止多轮强化学习中的预填充-解码干扰，GLM-5 采用了预填充-解码 (PD) 分离机制。 在多轮学习环境中，长前缀预填充操作十分频繁（例如对话历史、工具轨迹、代码上下文）。在 DP 注意力机制下，将预填充和解码操作混合在同一服务资源上会造成严重的干扰：大量的预填充操作可能会抢占或中断服务器上正在进行的解码操作，导致其他样本无法持续推进，并显著增加尾部延迟。因此，GLM-5 利用了 slime 的预填充-解码 (PD) 分离机制。通过在专用资源上运行预填充和解码操作，解码操作能够保持稳定且不间断，从而使长时域样本能够持续推进，并显著改善多轮智能体强化学习中的尾部行为。

### 部署鲁棒性：基于心跳的容错

大规模部署中，瞬态故障（例如，单个服务器崩溃、网络问题或性能下降）不可避免。GLM-5 利用 slime 的心跳驱动容错机制来确保此类事件下的训练连续性：部署服务器会定期发出由编排层监控的心跳信号，不健康的服务器会被主动终止并从推理路由器中注销。因此，重试请求会自动从故障或性能下降的服务器路由到健康的服务器，从而防止单个服务器事件中断部署，并保持端到端强化学习训练的连续性。

为了对智能体任务进行强化学习，我们设计了一个完全异步和解耦的强化学习基础设施，可以高效地处理长期智能体部署，并支持跨不同智能体框架的灵活多任务强化学习训练。

### 面向智能体训练的异步强化学习设计

由于部署过程具有长尾特性，朴素的同步强化学习训练会在部署阶段引入大量“气泡”，这是由于智能体任务的生成严重不平衡造成的，从而导致GPU长时间闲置。为了提高训练吞吐量，我们采用完全异步的训练范式来提升智能体强化学习的GPU利用率和训练效率。具体来说，我们将训练引擎和推理引擎解耦到不同的GPU设备上。推理引擎持续生成轨迹。一旦生成的轨迹数量达到预设阈值，该批次轨迹就会被发送到训练引擎以更新模型。为了减少策略延迟并保持训练策略的一致性，部署引擎使用的模型权重会定期与训练引擎的权重同步。训练引擎更新模型参数并将新的权重推送回推理引擎。

梯度更新。虽然异步更新可以显著提高整体训练效率，但也意味着不同版本的模型可能会生成不同的轨迹，从而引入严重的离策略问题。由于权重更新会因展开策略的变化而考虑不同的优化问题，因此我们在推理引擎每次权重更新后都会重置优化器。

基于服务器的多任务训练设计。为了解决多任务强化学习中轨迹生成的异构性问题（不同任务通常依赖于不同的工具集和任务特定的部署逻辑），我们引入了一个基于服务器的多任务部署编排器，用于多任务强化学习训练。该组件旨在通过一个拥有多个已注册任务服务的中央编排器，确保slime强化学习训练框架与各种下游任务之间的无缝兼容性。具体来说，每个任务都以独立微服务的形式实现自身的部署和奖励逻辑，并注册到中央编排器进行管理和调度。在部署阶段，中央编排器控制每个任务的部署比例和生成速度，以实现跨任务的均衡数据采集。至关重要的是，我们将所有智能体任务的轨迹标准化为统一的消息列表表示。这使得复杂智能体框架（例如软件工程任务）能够进行联合训练，同时支持对异构工作负载进行集中式后处理和日志记录。这种设计将任务特定的逻辑与核心训练循环清晰地隔离开来，从而实现了与多任务强化学习训练的无缝集成。作为 GLM-5 训练基础设施的骨干，该协调器支持超过 1000 个并发部署，并可自动、动态地调整任务采样率，以及对任务进度进行精细监控。

### 优化异步训练稳定性

Token-in-Token-out 与 Text-in-Text-out。在强化学习的展开式训练中，token-in-token-out (TITO) 指的是训练流程直接使用推理引擎生成的精确分词和解码后的词元流来构建学习轨迹。相比之下， text-in-text-out将展开式训练引擎视为一个黑盒，它返回最终的文本；训练器随后通过重新分词（通常还需要重新计算边界和截断）来重建轨迹，然后再计算损失。这种看似微小的选择却至关重要：重新分词可能会在词元边界、空格/归一化处理、截断或特殊词元放置等方面引入细微的不匹配，进而破坏动作与奖励/优势之间的步长对齐——尤其是在展开式训练被流式传输、截断或交错地分配给多个参与者时。我们发现，对于异步强化学习训练而言，token-in-token-out (TITO) 机制至关重要，因为它既能保持采样内容与优化内容之间精确的动作级对应关系，又能让参与者立即发出轨迹片段（token ID + 元数据），无需进行有损的文本往返传输，也无需等待学习器端进行事后重新分词。在实践中，我们实现了一个 TITO 网关，用于拦截所有来自部署任务的生成请求，并记录每条轨迹的 token ID 和元数据。这种设计将繁琐的 token ID 处理与下游代理部署逻辑隔离开来，同时避免了强化学习训练期间的重新分词不匹配。

剔除不符合策略和噪声较大的样本。在异步强化学习中，过长的轨迹可能严重偏离策略，从而导致训练不稳定。为了过滤掉这些严重偏离策略的样本，我们记录了生成时 rollout 引擎使用的策略权重版本。具体来说，对于每个响应，我们记录所涉及的模型版本序列。

此外，编码代理沙箱本身可能不稳定，并可能因与模型无关的原因（例如环境崩溃）而失效。此类失效会引入噪声训练信号，因为它们反映的是环境的不稳定性，而非模型的性能。为了缓解这种情况，我们会记录每个样本的失效原因，并排除因环境崩溃而失效的样本。对于基于组的采样方法（例如 GRPO），移除失效样本可能会导致组不完整。在这种情况下，如果有效样本的数量超过组大小的一半，我们会重复使用有效样本来填充组；否则，我们会丢弃整个组。此过程可以减少虚假奖励噪声并提高训练稳定性。

面向加速的DP感知路由。我们提出了一种面向数据并行（DP）的路由机制，用于在大规模多目标推理中保持键值缓存的局部性。在多轮代理工作负载中，来自同一轮次的顺序请求共享相同的前缀。为了最大化键值复用，我们强制执行轮次级亲和性：属于给定代理实例的所有请求都被路由到同一个 DP 排名。具体来说，我们引入了一个有状态的路由层，该路由层使用一致性哈希将每个轮次 ID 映射到一个固定的 DP 排名。这种映射在轮次之间保持稳定，从而消除了跨排名的缓存未命中。为了防止长期负载不均衡，我们将哈希与基于哈希空间的轻量级动态负载均衡相结合。这种设计避免了冗余的预填充计算，并且无需跨 DP 排名进行键值同步。随着轮次长度的增加，预填充成本与增量标记成正比，而不是与总上下文长度成正比。最终，我们实现了端到端延迟的降低和长上下文代理推理的有效吞吐量的提高。

### 搜索任务

针对深度搜索信息检索任务，我们构建了一个数据合成流程，用于生成具有挑战性的多跳问答对。每个问题都需要基于从多个网络资源聚合的证据进行多步骤推理。

Web知识图谱（WKG）构建与问题生成。 我们从早期搜索代理的路径出发，收集并去重所有遇到的URL，最终保留了跨多个领域的超过两百万个高信息量网页。LLM（逻辑逻辑模型）执行语义解析，用于实体识别、噪声过滤和结构化信息提取。WKG持续更新，并利用下游验证信号，通过实体对齐、属性规范化、关系整合和语义一致性修正等方法进行优化。基于WKG，我们采样低频到中频实体作为种子节点，并扩展其多跳邻域以形成完整的子图，同时控制扩展以减少重叠。利用针对高难度、多领域推理的提示，我们将每个子图转换为一个隐式编码多实体关系链的问题。

高难度问题过滤与验证。 我们采用三阶段流程来平衡难度和正确性：（1）移除无需工具的推理模型在八次独立尝试中至少一次能正确回答的问题。（2）过滤掉早期智能体只需几个步骤即可通过基本搜索、浏览和计算解决的问题。（3）应用验证智能体进行双向验证：我们从第二阶段的搜索轨迹中收集候选答案，然后分别验证候选答案和标注的真实答案的一致性，剔除答案不唯一、证据不一致或标签错误的样本。最终得到高质量、高难度且可靠的多跳问答对。

### 幻灯片生成

我们采用一种自改进流程，旨在通过强化学习和拒绝采样微调训练一个专门的幻灯片生成专家，从而系统地提升幻灯片生成性能。首先，我们使用监督微调（SFT）初始化模型，使其具备基本的幻灯片生成能力；然后，我们基于演示幻灯片的常见美学和结构特征，采用多级奖励机制进行强化学习。这一阶段显著提升了幻灯片生成质量。接下来，我们进行拒绝采样微调和掩码微调，将强化学习过程中获得的知识注入到训练语料库中。该流程以协调迭代的方式，共同提升了数据质量和模型能力。

我们提出了一种多级奖励机制，将基于 HTML 的幻灯片生成过程中的奖励信号划分为三个级别：

第一层：静态标记属性。此层侧重于生成的 HTML 中的声明性属性，包括定位、间距、颜色、字体、饱和度和其他样式属性。基于专业设计原则，我们设计了一套规则来规范模型在生成此类声明时的行为。这些规则确保生成的 HTML 的语法可解析性，同时将标记层的设计空间限制在一个针对表现力、结构清晰度、视觉和谐性和可读性进行优化的子空间内。此外，我们还引入了幻觉图像和重复图像检测机制，以抑制幻觉或冗余的图像。

第二层：运行时渲染属性。与静态检查不同，此层在渲染过程中评估 DOM 节点的运行时属性，例如元素宽度和高度、边界框以及其他几何布局指标。通过约束这些属性，我们鼓励生成的幻灯片在空间组织方面更贴近人类的审美偏好。我们开发了一种分布式渲染服务，能够在提取所需运行时属性的同时，以高吞吐量执行渲染任务。在训练过程中，我们观察到几种形式的奖励作弊行为，例如对过长内容进行硬截断或过度操纵间距（参见图 9）。为了缓解这些问题，我们改进了渲染器的实现，消除了可利用的漏洞，确保奖励信号真正激励的是美学上连贯的布局，而不是对几何指标的表面符合。

第三层：视觉感知特征。除了运行时渲染的限制之外，我们还纳入了对渲染幻灯片的感知层面评估。例如，我们将异常的空白模式作为辅助信号进行检测，以进一步提升整体构图平衡和视觉美感。

训练策略。在强化学习过程中，这些信号会被联合优化，以提高生成的 HTML 的结构有效性、增强布局组织并提升整体视觉美感。除了奖励设计之外，我们还通过动态采样来重塑训练分布。具体来说，我们会有概率地丢弃一部分结构简单的样本，从而使优化能够专注于更具挑战性的页面，并提高在复杂组合场景下的鲁棒性。我们还采用了标记级策略梯度损失来稳定优化过程 。57此外，我们引入了一种平衡策略，将同一样本的不同推广结果分布到多个训练批次中，从而减少优化偏差并提高训练稳定性。

拒绝抽样。在拒绝采样阶段，强化学习中使用的奖励函数被转移到数据过滤流程中，以构建高质量的训练子集。在页面级别，过滤标准包括代码有效性和编译可行性。在轨迹级别，我们进一步强制执行工具执行正确性和全局内容多样性约束，以确保结构一致性。我们采用 bset-N 这种选择策略从多个独立生成的候选样本中保留质量最高的样本。该机制有效地重新调整了样本分布的权重，使其更倾向于高质量实例，从而提高了样本效率并增强了训练稳定性。

基于mask的精细化。尽管拒绝采样能够去除大部分低质量输出，但某些轨迹中仍存在缺陷，这些缺陷仅存在于少数页面中。丢弃此类样本会降低有效数据利用率并增加生成成本。为了解决这个问题，我们引入了一种基于掩码的校正机制，该机制能够自动识别缺陷页面并应用掩码，同时保留同一轨迹中的高质量内容。这种选择性优化保留了有价值的监督信号，提高了有效数据利用率，并减少了冗余的重新生成开销，从而提升了整体训练效率。

经验改进。严格符合 16:9 宽高比的生成页面比例从 40% 提升至 92%，同时页面溢出情况显著减少。人工评估进一步表明，与 GLM-4.5 相比，GLM-5 在内容质量、布局合理性和视觉美感方面的胜率分别达到 60%、57.5% 和 65%，总体胜率达到 67.5%。这些结果为所提出的多级奖励机制和自改进框架的有效性提供了实证支持。
