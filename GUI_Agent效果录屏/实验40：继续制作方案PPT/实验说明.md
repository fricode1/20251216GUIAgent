# 实验步骤

## 从 MAI-UI 借鉴思路

用户交互能力，当指令不完整或表达模糊时主动提问，通过与用户交互先行澄清关键信息、拒绝臆断，确保执行路径始终与用户的真实意图对齐。

这一“主动提问”的能力，靠 ask_user 动作实现。当模型判断用户指令缺少关键信息，或存在歧义时，它会优先输出“提问”动作，而不是强行操作 UI。

四、明确技术方案
完成状态：已完成
使用 Browser Use Agent，而非 Computer Use Agent。
原因：Computer Use Agent 的任务执行成功率和执行速度远远不能满足产品化需求。Browser Use Agent 具有相对较高的任务执行成功率和相对较快的执行速度。多数公安业务平台基于Web开发，适用 Browser Use Agent。

五、明确开发方案

在浏览器中通过JS代码实现 Browser Use Agent，而非通过Python代码实现 Browser Use Agent。
原因：因为JS代码仅依赖浏览器环境，基层民警的电脑上均可运行，但很难安装python。
