5. 基于 Human in the Loop 的交互机制

Human-in-the-loop (HITL) 是一种将人类输入和专业知识融入人工智能系统生命周期的协作方法。本项目旨在利用 HITL 思想，将用户互动深度融入智能体工作流，通过主动交互补全信息、优化决策，解决真实场景中指令模糊、信息缺失的关键问题。

#### 核心设计逻辑

- 聚焦真实场景痛点：用户指令常存在歧义、关键信息缺失（如未提供收件人邮箱、任务参数不明确），仅靠模型推测易出错。
- 构建双向交互机制：让智能体具备 “主动询问” 能力，而非被动执行，形成 “用户指令→智能体澄清→用户补充→任务完成” 的闭环。

#### 具体实现方式

1. 交互触发机制
	- 智能体在任务执行中实时检测信息缺口，当发现指令模糊（如 “发送最近的简历” 未明确 “最近时间范围”“收件人”）或关键参数缺失时，自动触发 `ask_user` 动作。
	- 无需用户主动反馈，智能体主动生成精准的澄清问题，例如询问 “要查看的监控画面的对应点位是什么？”“需要布控的嫌疑人的身份证号是什么？”。
2. 多轮交互支持
	- 支持单轮或多轮澄清，直至获取完成任务所需的全部信息。
	- 交互过程中，智能体将用户回复记录至轨迹记忆，后续步骤自动复用补充信息，无需重复询问。
3. 与任务流程深度融合
	1. 交互动作与 UI 操作并列纳入智能体动作空间，成为任务执行的必要环节。
	2. 澄清后自动 Resume 任务流程，例如获取摄像头点位名称后直接完成监控画面播放，无需用户额外操作。

#### 核心价值

1. 提升任务成功率：通过补全信息避免因指令模糊导致的执行偏差
2. 增强实用性：适配真实场景中不规范、不完整的用户指令，降低用户使用门槛，无需刻意提供详细参数。
3. 优化用户体验：主动澄清替代 “沉默失败”，让用户明确模型执行逻辑，提升交互透明度。

## 制作一页《特定业务平台的SOP流程知识库构建：技术路线》PPT

### 必要性

专业业务平台具有高封闭性与复杂逻辑。Web Agent 拥有通用知识，但无法操作专业性强的特定系统。必须通过结构化的实操经验，将通用 Web Agent 转化为“行业专家”。
### SOP

一份标准操作程序（Standard Operating Procedure, SOP）是一种结构化的文档，提供清晰的、逐步的指令，用于完成特定流程或任务。
### 具体技术路线（4条路径结合的方式）

-  编制任务的SOP（这就是我目前采用的方式）：优点：能够快速完成数个指定任务。缺点：扩展性差。
- 现有说明书+RAG。优点：说明书里全面介绍了系统的各项功能。缺点：不是每个平台都能找到说明书。
- 专家操作一遍，后台录制键鼠轨迹，LLM 解析轨迹生成 SOP，作为后续类似任务的参考
	- 优点：适合重复性任务。
- 自主探索，学习两个页面的跳转逻辑。
