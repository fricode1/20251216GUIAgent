# 实验步骤

## 描述步骤

假设市区禁止渣土车行驶。为了找到在市区行驶的渣土车，在MRAG平台进行布控，能够收集市区监控中出现渣土车的画面。

如果MRAG平台捕获了渣土车画面，需要到视综平台调取该违规车辆的车主信息及其联系方式。

希望靠 Web Agent 自主执行整个流程。

## 互联网模拟步骤

上述应用是内网的，不方面调试。为了调试，需要在互联网模拟类似场景。

在百度图像中搜索黄晓明图像。在豆包中让AI解释这幅图像。

### 第一次尝试

- 工具：nanobrowser
- prompt：在百度图像中搜索黄晓明图像。在豆包中让AI解释这幅图像。
- 起始页：aistudio.google.com
- 结果：Navigation failed: Could not parse navigator response
- 反思：可能由于起始页太复杂了

### 第二次尝试

- 工具：nanobrowser
- prompt：在百度图像中搜索黄晓明图像。在豆包中让AI解释这幅图像。
- 起始页：image.baidu.com
- 结果：在豆包页面，planner无法解析deepseek的返回值
- 反思：单独在豆包页面测试nanobrowser的可用性

### 第三次尝试

- 工具：nanobrowser
- prompt：搜索黄晓明
- 起始页：doubao.com
- 结果：AI不在豆包搜索，反而跳转到百度搜索
- 反思：强制要求AI在豆包进行查询

### 第四次尝试

- 工具：nanobrowser
- prompt：在当前页面的搜索框，输入黄晓明并点击搜索按钮
- 起始页：doubao.com
- 结果：Navigation failed: Could not parse navigator response
- 反思：换种问法

### 第五次尝试

- 工具：nanobrowser
- prompt：当前页面是一个chatbot。向该chatbot询问黄晓明的信息。
- 起始页：doubao.com
- 结果：向搜索框输入了消息，但是又消失了。无法发送信息
- 反思：换种问法

### 第六次尝试

- 工具：nanobrowser
- prompt：向当前页面的文本框输入 黄晓明，然后单击发送按钮
- 起始页：doubao.com
- 结果：文字是通过注入方式实现的，这导致无法唤起发送按钮，且文字一下就消失了
- 反思：看来使用注入方式操作网页是有局限性的。1. DOM很长，需要过滤。2. 只能执行网页操作，但是网页弹出文件管理器就束手无策了，而这是实际应用所需求的。3. 通过注入方式操作网页，有时无法像人类操作一样唤起网页正常反映。然而它也有独特的优点：安装使用方便。难道我要完全舍弃吗？还是在此基础上修补？如果是修补方案的话，如何解决上面三个问题：1. 目前已经实现了DOM过滤的功能。2. 弹出文件管理器的情况，让人类操作。3. 大多数情况下，注入是有效的。

### 第七次尝试

- 工具：nanobrowser
- prompt：在百度图像中搜索黄晓明图像。将图像上传至 https://www.watermarkremover.io/zh
- 起始页：image.baidu.com
- 结果：1. Planning failed: Failed to invoke deepseek-chat with structured output 2. 明明下载成功了，但是agent不知道自己成功了，不停重复下载。
- 反思：研究下浏览器能否判断本地文件是否存在。因为我知道在浏览器中输入 C:/Users/ 能够看到本地文件

### 研究浏览器能否能与本地文件进行交互

结论：
1. 在浏览器中输入 C:/Users/ 能够看到本地文件
2. 浏览器能够下载文件到本地
3. 让 nanobrowser/browseruse 描述路径为 C:/Users/ 标签页的内容，失败。BrowserOS，成功。

# 实验结果

一定要简化、分解任务。因为想要把产品做好，需要很多时间去打磨。不可能一步到位。
- 如果想执行跨平台任务，一定要确保在单平台上的子任务能够成功执行。
- 如果想执行跨平台任务，一定要先用文本传递，而不是图像传递。
- 一定要先实现可操作性，再追求意义。
