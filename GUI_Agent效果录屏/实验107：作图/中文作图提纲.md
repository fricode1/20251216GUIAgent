多模态大模型架构绘图大纲

一、整体布局结构
┌─────────────────────────────────────────────────────────────────────────────┐
│  第一层：多模态输入层 (顶部横向排列)                                            │
│  第二层：Tokenizer分词编码层 (承接输入层)                                       │
│  第三层：统一Token序列 (中央枢纽)                                               │
│  第四层：三大子模型并行处理 (左中右三分布局)                                     │
│  第五层：各子模型内部详细模块 (垂直展开)                                         │
│  第六层：输出层 (底部汇总)                                                      │
└─────────────────────────────────────────────────────────────────────────────┘

二、第一层：多模态输入层
位置：最顶部，横向三等分
元素：
  ├─ 左框：文本输入 (Text)
  │     标注：自然语言文本、问题描述、指令提示
  │
  ├─ 中框：图像输入 (Image)  
  │     标注：静态图片、视觉文档、版面图像
  │
  └─ 右框：视频输入 (Video)
        标注：动态视频序列、时序视觉数据

连接线：三个框分别向下引出箭头 → 汇聚到Tokenizer层

三、第二层：Tokenizer分词编码层
位置：输入层正下方，横向贯穿的大框
内部结构（左中右排列）：
  ├─ 左侧：文本分词器
  │     输出标注：→ 文本Tokens
  │
  ├─ 中部：图像编码器 (MoonViT-3D)
  │     关键技术标注：
  │       • NaViT Patch打包策略
  │       • 原生分辨率处理
  │     输出标注：→ 视觉Tokens
  │
  └─ 右侧：视频编码器 (3D ViT压缩)
        关键技术标注：
          • 4帧分组时空压缩
          • 时间池化特征聚合
        输出标注：→ 时空Tokens

连接线：三种Tokens向下汇聚 → 统一Token序列

四、第三层：统一Token序列（中央枢纽）
位置：Tokenizer层正下方，居中突出显示
内容框：
  内部文本：[文本Tokens] + [视觉Tokens] + [模态分隔符] + [PAD锚点]
  
连接线：从该层向下分出三条主箭头 → 分别指向左(理解)/中(表征)/右(生成)三个子模型

五、第四层：三大子模型标题区（横向三分）
位置：统一Token序列正下方，横向三等分大标题框

左框标题：
  多模态理解模型 (Understanding Model)
  基座：Kimi K2 MoE
  输出类型：文本输出

中框标题：
  多模态表征模型 (Embedding Model)  
  基座：Qwen3-VL
  输出类型：向量输出

右框标题：
  多模态生成模型 (Generation Model)
  基座：DiT / Qwen3
  输出类型：媒体输出

六、第五层A：理解模型内部详细结构（左侧垂直展开）
位于左框标题下方，垂直堆叠5个模块：

模块1：MoonViT-3D视觉编码器
  技术细节：
    • NaViT Patch打包策略（支持任意分辨率）
    • 3D ViT时空压缩（4帧分组处理）
    • 时间池化特征聚合
    • 图像/视频编码器权重共享
  输入：视觉Tokens
  输出：视觉特征 → 向下箭头

模块2：MLP投影器
  功能：视觉-语言特征空间对齐转换
  技术：多层感知机映射
  输入：视觉特征
  输出：对齐后的融合特征 → 向下箭头

模块3：Kimi K2 MoE语言模型（核心，加粗框）
  参数规模：
    • 总参数：1.04万亿
    • 激活参数：320亿
    • 专家单元：384个
    • 每Token激活：8个专家
    • 稀疏度：48%
  功能：跨模态推理与生成
  输入：融合特征+文本Tokens
  输出：推理结果 → 向下箭头

模块4：输出层
  内容：理解/推理/生成结果（文本形式）

连接线：模块1→模块2→模块3→模块4，垂直串联

七、第五层B：表征模型内部详细结构（中部垂直展开）
位于中框标题下方，垂直堆叠5个模块：

模块1：模态适配处理模块
  处理逻辑：
    ├─ 图像：保留原始宽高比 → 转张量格式
    ├─ 视频：固定帧率采样 + 最大帧数限制 → 保留时序特性
    └─ 文档：按版面拆分文字/图片/表格 → 按顺序拼接
  约束：32K序列长度上限

模块2：输入处理与模板解析模块
  技术：Qwen3-VL指令感知式上下文模板
  结构：
    ├─ 系统消息段：任务指令
    ├─ 用户消息段：多模态实例
    └─ PAD令牌：锚点作用
  输出：标准化令牌序列

模块3：基础支撑模块（双组件并行）
  组件A：Qwen3-VL视觉编码器 → 视觉Tokens特征提取
  组件B：Qwen3 LM稠密解码器 → 跨模态语义融合
  机制：因果注意力机制
  能力：32K超长序列、30+种语言

模块4：核心编码模块（双编码器架构）
  处理流程：
    ├─ 文本模态：语言解码器直接编码
    ├─ 视觉模态：视觉编码器→语言解码器编码
    └─ 多模态组合：各模态独立编码→跨模态轻量融合
  输出：高维特征序列

模块5：嵌入向量生成模块（输出层）
  机制：以PAD令牌为锚点，提取对应隐藏状态
  规格：
    • 2B版本：2048维固定向量
    • 8B版本：4096维固定向量
  特性：维度与输入模态/长度无关
  应用：余弦相似度度量、多模态检索

连接线：模块1→模块2→模块3→模块4→模块5，垂直串联

八、第五层C：生成模型内部详细结构（右侧垂直展开，分两大分支）
位于右框标题下方，先分两个子标题框，再各自垂直展开

子标题左：视图生成大模型（MMDiT双分支架构）
子标题右：语音生成大模型（双轨自回归架构）

【视图生成分支详细模块】（垂直5层）

模块1：文本/图像编码器（输入层）
  功能A：文本提示词语义编码 → 标准化特征向量
  功能B：参考图像视觉特征提取（图像引导任务）

模块2：双分支并行处理（左右并排子框）
  左子框-视觉分支处理模块（MMDiT）：
    • 画面主体构建
    • 动作动态建模
    • 镜头运动设计
    • 场景过渡处理
    → 内置听觉分支交互接口
  
  右子框-听觉分支处理模块（MMDiT）：
    • 人声生成
    • 音效设计
    • 背景音乐
    • 语音韵律捕捉（语言/方言）
    → 内置视觉分支交互接口

模块3：跨模态联合模块（中央融合）
  机制：跨模态注意力机制
  功能：
    • 视觉-听觉特征深度交互
    • 时间维度匹配优化
    • 语义维度协同校准
  目标：保障口型/动作/音效精准同步

模块4：音视频联合生成模块（DIT）
  技术：Diffusion Transformer生成
  输入：统一跨模态特征表征
  输出：
    • 视频帧序列
    • 音频波形数据

模块5：输出层
  内容：音视频内容（时间同步、语义一致）

连接线：模块1→模块2（分两支）→模块3（汇聚）→模块4→模块5

【语音生成分支详细模块】（垂直8层）

模块1：文本分词器
  功能：自然语言文本 → 文本Tokens

模块2：说话人编码器（可学习模块）
  功能：从参考语音提取
    • 音色特征
    • 语调特征
  应用：语音克隆、特定说话人生成

模块3：语音分词器（双版本并行）
  版本A-25Hz单码本：
    • 语义+声学融合编码
    • Qwen2-Audio编码器
    • 块级扩散转换器流式重建
  
  版本B-12Hz多码本：
    • 语义-声学解纠缠量化
    • 12.5Hz多层多码本
    • 轻量级因果卷积网络
    • 毫秒级首包延迟

模块4：双轨表示融合模块
  操作：文本Tokens + 声学Tokens → 通道维度拼接
  输出：融合特征表示

模块5：Qwen3语言模型（骨干网络）
  能力：高并发、低延迟推理
  功能：根据文本Tokens实时预测声学Tokens序列
  特殊：12Hz版本优先预测第0码本

模块6：多令牌预测模块（12Hz版本专用）
  输入：第0码本特征
  输出：所有残留码本特征
  机制：单帧即时生成
  目标：降低端到端延迟、提升表达一致性

模块7：Code2Wav模块（双版本）
  版本A-适配25Hz：
    • 块级扩散转换器
    • 改进型BigVGAN声码器
    • 流匹配技术高保真重建
  
  版本B-适配12Hz：
    • 纯左上下文流式编解码器
    • 无需等待未来上下文
    • 增量音频重建

模块8：流式解码器（输出层）
  版本A-25Hz：
    • 滑动窗口块注意力机制
    • 限制上下文参考范围
    • 长序列稳定流式生成
  
  版本B-12Hz：
    • 全因果特征编解码器
    • 12.5Hz速率逐帧处理
    • 满足实时在线服务

模块9：输出层
  内容：实时语音输出
  能力：多语言生成、语音克隆、细粒度控制

连接线：模块1/2/3→模块4→模块5→模块6→模块7→模块8→模块9（垂直串联，模块6为12Hz专用旁支）

九、第六层：输出汇总层（底部）
位置：最底部，横向三等分对应三个子模型

左框：理解模型输出
  内容：文本形式的理解/推理/生成结果

中框：表征模型输出  
  内容：固定维度稠密向量（2048D/4096D）

右框：生成模型输出
  内容：
    ├─ 视图分支：视频帧 + 音频波形（同步音视频）
    └─ 语音分支：实时语音流
